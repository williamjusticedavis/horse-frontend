# React Template

A production-ready React + TypeScript frontend starter. Clone it and start building — all foundations are laid.

## Stack

| Layer           | Library                                                                       |
| --------------- | ----------------------------------------------------------------------------- |
| Runtime & tests | [Bun](https://bun.sh)                                                         |
| UI              | [React 19](https://react.dev) + TypeScript                                    |
| Build           | [Vite 6](https://vitejs.dev)                                                  |
| Routing         | [TanStack Router v1](https://tanstack.com/router) (file-based)                |
| Data fetching   | [TanStack Query v5](https://tanstack.com/query)                               |
| Styling         | [Tailwind CSS v4](https://tailwindcss.com)                                    |
| Icons           | [Lucide React](https://lucide.dev)                                            |
| Toasts          | [Sonner](https://sonner.emilkowal.ski)                                        |
| Modals          | [Radix UI Dialog](https://www.radix-ui.com/primitives/docs/components/dialog) |
| Linting         | ESLint 9 (flat config) + TypeScript ESLint                                    |
| Formatting      | Prettier + prettier-plugin-tailwindcss                                        |

## Getting started

```bash
# 1. Clone
git clone <repo-url> my-app && cd my-app

# 2. Install
bun install

# 3. Set environment variables (optional — defaults work out of the box)
cp .env.example .env

# 4. Start dev server
bun run dev
```

Open [http://localhost:5173](http://localhost:5173).

## Scripts

```bash
bun run dev          # Start Vite dev server (also regenerates routeTree.gen.ts)
bun run build        # Type-check + Vite production build
bun run preview      # Preview the production build locally
bun run lint         # ESLint
bun run lint:fix     # ESLint with auto-fix
bun run format       # Prettier (write)
bun run format:check # Prettier (check only)
bun run test         # Bun test runner
bun run test:watch   # Tests in watch mode
bun run typecheck    # tsc --noEmit
```

## Environment variables

All variables are optional — the defaults work without a `.env` file.

| Variable         | Default (dev)           | Default (prod)     | Description                                            |
| ---------------- | ----------------------- | ------------------ | ------------------------------------------------------ |
| `VITE_API_URL`   | `http://localhost:8000` | `''` (same origin) | Backend API base URL used by `src/lib/api.ts`          |
| `VITE_BASE_PATH` | `/`                     | `/`                | URL sub-path the app is served from (e.g. `/template`) |

`src/lib/config.ts` reads these at build time and exports them as `config.apiUrl` and `config.basePath`. Import from there instead of reading `import.meta.env` directly.

## Dark mode

Handled entirely in `src/context/theme-context.tsx`. The preference is persisted to `localStorage`. The `.dark` class is toggled on `<html>` which triggers Tailwind's `@variant dark` overrides. The Navbar renders a toggle button — click the sun/moon icon.

## Adding routes

1. Create a file in `src/routes/`, e.g. `src/routes/settings.tsx`
2. Export a `Route` using `createFileRoute`:

```tsx
import { createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/settings')({
  component: () => <div>Settings page</div>,
})
```

3. Run `bun run dev` — the router plugin regenerates `routeTree.gen.ts` automatically.

## Fetching data

```tsx
import { useQuery } from '@tanstack/react-query'
import { api } from '@/lib/api'

type User = { id: number; name: string }

function Profile({ id }: { id: number }) {
  const { data, isPending, error } = useQuery({
    queryKey: ['user', id],
    queryFn: () => api.get<User>(`/users/${id}`),
  })

  if (isPending) return <Spinner />
  if (error) return <p>Error: {error.message}</p>
  return <p>Hello, {data.name}</p>
}
```

## Theming

Colors are CSS custom properties (HSL channels) defined in `src/styles/globals.css`. Override them in `:root` (light) and `.dark` for instant theme changes. Tailwind picks them up via `@theme inline { --color-background: hsl(var(--background)); ... }`.

To change the primary colour, edit `--primary` and `--primary-foreground` in both `:root` and `.dark`.

## Notes

- `routeTree.gen.ts` is committed to the repo so TypeScript is happy before the first `bun run dev`. It is regenerated automatically whenever you add/remove route files.
- React Query Devtools and TanStack Router Devtools are lazy-loaded and excluded from production builds automatically.
- Bun's test runner does not support `environment = "happy-dom"` in `bunfig.toml` on current versions. Instead, `setup-dom.ts` manually registers browser globals using happy-dom's `Window` class as preload 1 — this ensures the DOM exists before `@testing-library/dom` is imported.
